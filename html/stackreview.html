Introduction

I'm interested in stack architectures as they seem to be a good way of organising the memory aspect of temporal computing. There's a lot of similaritiues between a stack and time, not least time operates as a contiuouly popping stack with strict access constraints, so as a first approximation their similaritiues are worth exploring. I have the following questions:

  1. Are stack architectures universal?
  2. Are stack architecrtures efficient?
  3. Are there things that stack architecrures do that time architectures can't?
  4. What happens when you have more than one stack, does this make things more efficient?
  5. WHat has been built with thiese models and can I replicate
